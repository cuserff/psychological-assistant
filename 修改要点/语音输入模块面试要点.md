# 语音输入模块 —— 面试要点

## 一、为什么要做这个功能？

心理咨询场景下，很多用户（尤其情绪低落时）不愿意打字，语音输入可以**降低表达门槛**，让用户更自然地倾诉。项目原有 `useSpeech.js` 和 `VoiceRecorder.vue` 两个占位文件，需要将其实现。

## 二、技术选型

| 方案 | 优点 | 缺点 | 是否采用 |
|------|------|------|----------|
| **浏览器 Web Speech API** | 零成本、无后端、实时识别 | 仅 Chrome/Edge 支持良好 | **是** |
| 第三方语音服务（讯飞/百度） | 识别准确率更高、兼容性好 | 需后端中转、增加成本和复杂度 | 否 |

选择 Web Speech API 的理由：
- 项目是学习/演示项目，Chrome/Edge 覆盖率足够
- 无需额外后端接口和 API Key
- 支持实时中间结果，用户边说边看到文字

## 三、核心实现

### 1. useSpeech 组合函数（Composable 模式）

```javascript
export function useSpeech() {
  const SpeechRecognition = window.SpeechRecognition || window.webkitSpeechRecognition

  const isSupported = ref(!!SpeechRecognition)  // 浏览器兼容性检测
  const isListening = ref(false)                 // 录音状态
  const transcript = ref('')                     // 识别文字

  // 创建识别实例
  recognition = new SpeechRecognition()
  recognition.lang = 'zh-CN'          // 中文识别
  recognition.continuous = true        // 连续识别，不会一句话后停止
  recognition.interimResults = true    // 返回中间结果，实时更新

  recognition.onresult = (event) => {
    // 区分 isFinal（最终结果）和中间结果，拼接显示
    let finalText = '', interimText = ''
    for (let i = 0; i < event.results.length; i++) {
      if (event.results[i].isFinal) finalText += event.results[i][0].transcript
      else interimText += event.results[i][0].transcript
    }
    transcript.value = finalText + interimText
  }

  return { isSupported, isListening, transcript, startListening, stopListening }
}
```

**为什么用 Composable 而不是写在组件里？**
- 关注点分离：语音识别逻辑独立于 UI，可被任何组件复用
- 可测试性：纯逻辑函数更容易单元测试
- 符合 Vue 3 Composition API 的最佳实践

### 2. VoiceRecorder 组件

```html
<!-- 浏览器不支持时整个组件不渲染 -->
<div v-if="isSupported" class="voice-recorder">
  <button :class="{ listening: isListening }" @click="toggleListening">
    <svg><!-- 麦克风图标 --></svg>
  </button>
  <span v-if="isListening" class="listening-hint">
    <span class="pulse-dot"></span> 正在聆听...
  </span>
</div>
```

**组件通信方式**：停止录音后通过 `emit('result', text)` 将文字传给父组件：

```javascript
watch(isListening, (listening) => {
  if (!listening && transcript.value.trim()) {
    emit('result', transcript.value.trim())
  }
})
```

**为什么在 watch 停止时 emit，而不是实时 emit？**
- 中间结果不稳定，会频繁变化（"你好" → "你好吗" → "你好吗今天"）
- 等最终结果确定后一次性传给输入框，避免输入框内容闪烁
- 用户可以在发送前编辑修正文字

### 3. Chat 页面集成

```html
<div class="input-actions">
  <VoiceRecorder @result="handleVoiceResult" />
  <el-button @click="handleSend">发送</el-button>
</div>
```

```javascript
function handleVoiceResult(text) {
  inputMessage.value += text  // 追加而非覆盖，支持多次语音输入

```

## 四、交互设计细节

| 状态 | UI 表现 |
|------|---------|
| 默认 | 灰色麦克风图标按钮 |
| hover | 蓝色边框 + 浅蓝背景 |
| 录音中 | 红色按钮 + 脉冲动画（box-shadow） + "正在聆听..." |
| 浏览器不支持 | 按钮完全不渲染（`v-if="isSupported"`） |

**录音中的脉冲动画**：用 CSS `@keyframes` 实现 box-shadow 扩散效果，无需额外依赖：
```css
@keyframes pulse-ring {
  0%   { box-shadow: 0 0 0 0 rgba(239, 68, 68, 0.4); }
  70%  { box-shadow: 0 0 0 8px rgba(239, 68, 68, 0); }
  100% { box-shadow: 0 0 0 0 rgba(239, 68, 68, 0); }
}
```

## 五、关键技术点

### Web Speech API 配置三要素
- `lang: 'zh-CN'`：指定中文识别，不设置则跟随浏览器语言
- `continuous: true`：连续识别模式，用户说多句话不会自动断开
- `interimResults: true`：返回中间结果，实现"边说边显示"

### 生命周期管理
```javascript
onBeforeUnmount(() => {
  stopListening()  // 组件卸载时自动停止录音，防止内存泄漏
})
```

### 兼容性处理
```javascript
const SpeechRecognition = window.SpeechRecognition || window.webkitSpeechRecognition
```
Chrome 使用 `webkitSpeechRecognition` 前缀，标准 API 为 `SpeechRecognition`，需要两者兼容。
